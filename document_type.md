# 문서 유형

## 1. 행정/공문서

### 1-1. 공문

- 구조
    - 제목 → 수신/참조 → 본문 → 첨부 → 발신
- 특징
    - 형식 고정 (기관 표준 존재)
    - 법적/공식 효력 존재
- 포맷
    - HWP, DOCX, PDF

### 1-2. 보고서 (행정/업무)

- 구조
    - 개요 → 목적 → 현황 → 분석 → 결과 → 결론
- 특징
    - 데이터 기반, 논리적 흐름 중요
    - 내부 의사결정 및 업무 보고 목적
- 포맷
    - PPTX, DOCX, PDF

### 1-3. 기안서

- 구조
    - 제목 → 기안 목적 → 내용 → 기대 효과 → 결재 라인
- 특징
    - 의사결정 요청 문서
- 포맷
    - 그룹웨어 양식, DOCX

### 1-4. 결재문서

- 구조
    - 기안 내용 + 결재선 + 승인/반려 이력
- 특징
    - 전자결재 시스템 기반
- 포맷
    - 전자결재 시스템 (HTML/XML 기반)

---

## 2. 비즈니스 문서

### 2-1. 제안서

- 구조
    - 문제 정의 → 해결 방안 → 기술/서비스 → 비용 → 일정
- 특징
    - 설득 중심 (고객 관점)
- 포맷
    - PPTX, PDF

### 2-2. 사업계획서

- 구조
    - 시장 분석 → 비즈니스 모델 → 전략 → 재무 계획
- 특징
    - 투자/사업 검토용
- 포맷
    - PPTX, DOCX, PDF

### 2-3. 회의록

- 구조
    - 회의 정보 → 논의 내용 → 결정 사항 → 액션 아이템
- 특징
    - 추적성과 책임 명확화
- 포맷
    - DOCX, Notion, Markdown

### 2-4. 이메일

- 구조
    - 제목 → 인사 → 본문 → 요청/결론 → 서명
- 특징
    - 빠른 커뮤니케이션
- 포맷
    - HTML, Plain Text

---

## 3. 기술 문서

### 3-1. API 명세서

- 구조
    - Endpoint → Method → Request → Response → Error
- 특징
    - 인터페이스 계약 (Contract)
- 포맷
    - OpenAPI (YAML/JSON), Markdown

### 3-2. 설계서

- 구조
    - 시스템 개요 → 아키텍처 → DB → 흐름도 → 인터페이스
- 특징
    - 시스템 전체 구조 설명
- 포맷
    - DOCX, PPTX, Markdown + Diagram (Draw.io, Figma)

### 3-3. 기술 스펙 문서

- 구조
    - 요구 사항 → 제약 조건 → 기술 선택 → 상세 설계
- 특징
    - 구현 기준 정의
- 포맷
    - Markdown, DOCX

### 3-4. 매뉴얼/가이드

- 구조
    - 개요 → 설치/설정 → 기능 설명 → 절차 → FAQ → 문제 해결
- 특징
    - 사용자 대상 단계별 안내
    - 스크린샷/다이어그램 포함 빈도 높음
- 포맷
    - PDF, DOCX, HTML, Markdown

---

## 4. 계약/법률 문서

### 4-1. 계약서

- 구조
    - 당사자 → 목적 → 조건 → 책임 → 분쟁 해결
- 특징
    - 법적 구속력
- 포맷
    - DOCX, PDF

### 4-2. 약관

- 구조
    - 정의 → 서비스 내용 → 이용 조건 → 책임 제한
- 특징
    - 사용자 대상 법적 고지
- 포맷
    - HTML, PDF

### 4-3. NDA (비밀 유지 계약)

- 구조
    - 비밀 정보 정의 → 보호 의무 → 기간 → 예외
- 특징
    - 정보 보호 목적
- 포맷
    - DOCX, PDF

---

## 5. 기록 문서

### 5-1. 로그

- 구조
    - Timestamp → Level → Message → Context
- 특징
    - 시스템 디버깅 핵심
- 포맷
    - TXT, JSON, 로그 시스템 (ELK)

### 5-2. 일지

- 구조
    - 날짜 → 작업 내용 → 이슈 → 결과
- 특징
    - 업무 추적
- 포맷
    - Markdown, Notion

### 5-3. 감사 기록

- 구조
    - 사용자 → 행위 → 시간 → 변경 내용
- 특징
    - 보안/감사 대응
- 포맷
    - DB, JSON

### 5-4. 히스토리

- 구조
    - 변경 전/후 → 작성자 → 시간 → 이유
- 특징
    - 버전 관리
- 포맷
    - Git, DB, Markdown

---

## 6. 학술/연구 문서

### 6-1. 논문

- 구조 (IMRAD 표준)
    - Introduction → Methods → Results → Discussion
- 특징
    - 재현성 + 근거 중심
- 포맷
    - PDF, LaTeX

### 6-2. 리포트 (학술/분석)

- 구조
    - 문제 → 분석 → 해결 → 결론
- 특징
    - 학습/분석 중심
    - 행정 보고서(1-2)와 구분: 학술적 분석·연구 목적
- 포맷
    - DOCX, PDF

### 6-3. 연구보고서

- 구조
    - 배경 → 방법 → 실험 → 결과 → 활용 방안
- 특징
    - 실무 + 연구 중간 형태
- 포맷
    - DOCX, PDF

---

## 7. 양식/서식 문서

### 7-1. 신청서

- 구조
    - 신청자 정보 → 신청 항목 → 사유 → 첨부 → 서명
- 특징
    - 정형화된 입력 필드 중심
    - 기관/조직별 표준 양식 존재
- 포맷
    - HWP, DOCX, PDF, HTML (웹 폼)

### 7-2. 설문지

- 구조
    - 안내문 → 질문 (객관식/주관식) → 응답란
- 특징
    - 데이터 수집 목적
- 포맷
    - Google Forms, DOCX, PDF

### 7-3. 체크리스트

- 구조
    - 점검 항목 → 기준 → 결과 (O/X) → 비고
- 특징
    - 점검/확인 목적, 반복 사용
- 포맷
    - XLSX, PDF, Markdown

---

## 8. 인사/HR 문서

### 8-1. 이력서/자기소개서

- 구조
    - 인적 사항 → 학력 → 경력 → 자격 → 자기소개
- 특징
    - 채용 프로세스 핵심 문서
- 포맷
    - DOCX, PDF, HTML

### 8-2. 채용공고

- 구조
    - 회사 소개 → 직무 → 자격 요건 → 우대 사항 → 접수 방법
- 특징
    - 외부 공개 대상
- 포맷
    - HTML, PDF, 채용 플랫폼

### 8-3. 인사평가서

- 구조
    - 평가 기간 → 성과 목표 → 달성도 → 역량 평가 → 종합
- 특징
    - 내부 기밀 문서, 정량+정성 혼합
- 포맷
    - 그룹웨어 양식, DOCX, XLSX

---

## 9. 데이터/분석 문서

### 9-1. 스프레드시트

- 구조
    - 헤더 → 데이터 행 → 수식/집계 → 시트 분리
- 특징
    - 테이블 중심, 수치 데이터 파싱 필요
- 포맷
    - XLSX, CSV, ODS

### 9-2. 대시보드 리포트

- 구조
    - KPI → 차트/그래프 → 요약 → 상세 데이터
- 특징
    - 시각화 중심, 정기 발행
- 포맷
    - PDF, HTML, BI 도구 (Tableau, Power BI)

### 9-3. 통계 보고서

- 구조
    - 조사 개요 → 표본 설계 → 결과표 → 해석
- 특징
    - 정량 데이터 기반, 표/도표 비중 높음
- 포맷
    - PDF, XLSX, DOCX

---

## 10. 홍보/마케팅 문서

### 10-1. 보도자료

- 구조
    - 헤드라인 → 부제 → 리드문 → 본문 → 문의처
- 특징
    - 역피라미드 구조, 객관적 톤
- 포맷
    - DOCX, PDF, HTML

### 10-2. 브로슈어/카탈로그

- 구조
    - 표지 → 제품/서비스 소개 → 사양 → 연락처
- 특징
    - 시각 디자인 비중 높음, 텍스트 추출 난이도 높음
- 포맷
    - PDF, AI (Adobe Illustrator), InDesign

### 10-3. 마케팅 기획서

- 구조
    - 시장 분석 → 타겟 → 전략 → 채널 → 예산 → KPI
- 특징
    - 내부 의사결정용
- 포맷
    - PPTX, DOCX, PDF
