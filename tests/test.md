## 테스트 1. 문단 내 문장 임베딩 유사도 (Docling 기반)

- 파일: `test_docling_similarity.py`
- 시각화: `similarity_heatmap_docling.png`
- 가설: "같은 섹션 내 문단 간 임베딩 유사도는 다른 섹션보다 높을 것이다"

### 결과

| 항목 | 값 |
|---|---|
| 같은 섹션 내 문단 유사도 | 0.4327 |
| 다른 섹션 간 문단 유사도 | 0.2914 |
| 차이 | +0.1413 |
| 경계 탐지 (비경계 - 경계) | +0.0747 |

### 해석

- 가설 지지: 같은 섹션 내 문단이 유의미하게 유사
- 다만 유사도 0.43은 "같은 주제" 치고 낮음 → 문서 전체가 같은 도메인(섬유 가공)이라 베이스라인 자체가 높기 때문
- Contextual Chunking(접두어)의 필요성 뒷받침

## 테스트 2. 문단별 임베딩 유사도 차이

- 파일: `test_paragraph_similarity.py`
- 시각화: `paragraph_similarity_test.png`
- 가설: "같은 섹션 내 인접 문단은 다른 섹션 인접 문단보다 유사도가 높다"

### 결과

| 항목 | 값 |
|---|---|
| 같은 섹션 인접 문단 유사도 | 0.4816 |
| 다른 섹션 인접 문단 유사도 | 0.4259 |
| 차이 | +0.0557 |
| p-value | 3.64e-21 |

### 해석

- 통계적으로 유의미(p < 0.001)하지만 분포 겹침이 큼
- 임베딩 유사도만으로 병합 판단하기엔 신호가 약함
- 의미: 문단 단위가 의미 단위를 반영하지만, 같은 문서 내에서는 구분력 제한적

## 테스트 3. 문단 Proposition 대표성

- 파일: `test_proposition_similarity.py`
- 시각화: `proposition_similarity_test.png`
- 가설: "문단의 Proposition 하나로 해당 문단의 의미를 대표할 수 있다"

### 결과

| 항목 | 값 |
|---|---|
| Proposition ↔ 문장 평균 유사도 | 0.6181 |
| 중앙값 | 0.6248 |
| 문단 내 표준편차 | 0.1053 |
| p-value (> 0.5 검정) | 6.52e-226 |

### 해석

- **가설 강하게 지지**: Proposition이 문단 내 문장들과 평균 0.62 유사도
- 표준편차 0.105 → 특정 문장만이 아니라 전체 문단을 균일하게 대표
- 유사도 낮은 하위 케이스는 LLM이 중국어로 응답한 경우 (모델 한계)
- **결론: Proposition 임베딩으로 검색 후 원본 문단 반환하는 구조 유효**

## 테스트 4. 섹션 Proposition 대표성

- 파일: `test_section_proposition.py`
- 시각화: `section_proposition_test.png`
- 가설: "섹션(소제목 단위)의 Proposition 하나로 섹션 내용을 대표할 수 있다"

### 결과

| 항목 | 값 |
|---|---|
| Proposition ↔ 문단 평균 유사도 | 0.5326 |
| 청크별 평균의 평균 | 0.6025 |
| 문단 내 표준편차 | 0.1115 |
| p-value (> 0.5 검정) | 8.97e-30 |

### 문단 수별 유사도

| 문단 수 | 섹션 수 | 유사도 |
|---|---|---|
| 2 | 209 | 0.7087 |
| 3 | 147 | 0.6488 |
| 4 | 85 | 0.6041 |
| 5 | 84 | 0.5747 |
| 6 | 58 | 0.5380 |
| 7 | 46 | 0.5145 |

- 문단 11개 이상: 0.44 이하로 하락 → 청크 크기 제어 필요

## 테스트 5. Multi-Proposition (1개 vs 2개)

- 파일: `test_multi_proposition.py`
- 시각화: `multi_proposition_test.png`
- 가설: "문단 5개 이상 섹션에서 Proposition 2개 추출 시 대표성이 향상된다"
- 대상: 문단 5개 이상 섹션 322개

### 결과

| 항목 | 1개 Prop | 2개 Prop | 차이 |
|---|---|---|---|
| 전체 평균 유사도 | 0.5120 | 0.5216 | +0.0096 |
| 개선된 섹션 비율 | - | - | 61.8% |
| p-value | - | - | 7.08e-03 |

### 해석

- 통계적으로 유의미(p=0.007)하지만 평균 개선폭 +0.01로 미미
- 38.2%에서 오히려 하락 (LLM 2개 분리 실패 케이스)
- **결론: LLM 호출 2배 대비 효과 부족 → Proposition 1개로 통일**

## 테스트 6. 패턴 기반 중목차 청킹

- 파일: `test_chunking_strategy.py`
- 시각화: `chunking_strategy_test.png`
- 가설: "Docling 후처리 → 패턴 기반 중목차 단위 청킹이 유효하다"

### 결과

| 항목 | 값 |
|---|---|
| Proposition ↔ 문단 평균 유사도 | 0.4000 |
| 청크별 평균의 평균 | 0.5284 |
| 청크 수 | 248개 |
| 적정 범위(512~2048) 비율 | 14.1% |

### 토큰 구간별 유사도

| 토큰 구간 | 청크 수 | 유사도 |
|---|---|---|
| < 200 | 121 | 0.5858 |
| 200~512 | 40 | 0.4966 |
| 512~1024 | 16 | 0.4484 |
| 1024~2048 | 19 | 0.4147 |
| 2048+ | 14 | 0.3696 |

### 해석

- 유사도 0.40으로 이전 테스트 대비 크게 하락
- **원인 1**: 패턴 기반 depth 분류가 실제 목차와 불일치 (248 vs 실제 13 중목차)
- **원인 2**: 합본형 문서에서 각 원고 내부 번호가 재시작 → 중목차/소목차 구분 불가
- **결론: 패턴 매칭이 아닌 목차 페이지 파싱으로 실제 계층 확정 필요**

5. **토큰 범위**: 최소 512, 적정 1024, 최대 2048
